## プロジェクト概要

学校と「複業で先生をしたい」外部人材をつなぐ、教育特化型マッチングプラットフォーム『複業先生』の開発に、2021年6月からプロダクト立ち上げ当初より業務委託エンジニアとして参画。教員不足・多様な学びのニーズという社会課題に向き合うEdTechスタートアップで、0→1フェーズの初期実装から継続的な機能開発・チームマネジメントまで幅広く担っている。

## 課題

- プロダクト黎明期における技術基盤の構築と、スケールを見据えたアーキテクチャ設計
- 学校が外部人材を探し・依頼するフローがアナログで、教員側の負担が大きかった
- 「授業の募集」（学校側が授業枠を公開し外部人材を公募する）機能が未整備で、マッチングの手段が一方向に偏っていた
- プロダクト拡大に伴うチームの成長と、開発フロー・品質管理の仕組みづくり

## アプローチ

立ち上げ当初から参画し、初期の技術選定・実装に携わった。以降も機能開発の中心として継続的に関与しており、マッチングフローの改善・ユーザー体験の向上・管理機能の整備など、プロダクト全体にわたって数多くの企画・実装を担当してきた。

なかでも「授業の募集」領域は企画段階から深く関与し、リリース後も応募数向上を目指したチューニングを継続している。

フロントエンドは Next.js + TypeScript、バックエンドは Ruby on Rails の API を拡張、インフラは AWS を採用。現在はエンジニアチームの一部マネジメントも担い、OKRを活用した目標管理・タスク管理・レビュー・オンボーディング支援まで対応している。

## 成果

- **複業先生登録数**: 2,500名規模（0→1フェーズから継続的に成長を牽引）
- **導入学校数**: 350校
- **利用生徒数**: 40,000名
- **授業満足度**: 98.5%
- 「授業の募集」機能を0から企画・実装し、リリース後も応募数改善のPDCAを継続
- OKRを導入したチームマネジメントで、開発の方向性と優先度を整理・運用
