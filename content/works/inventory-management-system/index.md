## プロジェクト概要

従業員50名規模の製造業クライアントが、長年Excelと紙帳票で運用していた在庫管理業務をシステム化するプロジェクトです。

現場担当者のオペレーション変更を最小限に抑えつつ、リアルタイムでの在庫把握・発注アラート・分析レポートを実現しました。

## 課題

### 業務課題
- 複数のExcelファイルが乱立し、データの一元管理ができていない
- 在庫確認のたびに倉庫担当者に電話連絡が必要
- 月次在庫棚卸に2日間かかっており、通常業務がストップ
- 欠品・過剰在庫の判断が担当者の経験則に依存

### 技術課題
- 既存の基幹システム（販売管理）との連携が必要
- バーコードスキャナーとの連携（既存設備を流用）
- 製造・仕入・販売の3部門が同時利用するため、権限管理が複雑

## アプローチ

### 経営企画フェーズ（要件定義）

システム開発の前に2週間かけて業務フローを整理しました：

1. **現場ヒアリング**: 倉庫担当者・購買担当者・経営者それぞれの視点で課題を収集
2. **業務フロー可視化**: AS-IS（現状）→ TO-BE（あるべき姿）を整理
3. **ROI試算**: 棚卸工数削減・欠品ロス削減・過剰在庫削減の金額換算
4. **フェーズ分け**: MVP（3ヶ月）→ 本格稼働（+2ヶ月）の段階的リリース計画策定

### システム設計

```
バックエンド: Ruby on Rails 7（API mode）
フロントエンド: React 18 + TypeScript + shadcn/ui
データベース: PostgreSQL（RDS）
インフラ: AWS（ECS + Fargate + RDS + S3）
CI/CD: GitHub Actions → ECR → ECS Blue/Green Deploy
```

### ポイント設計

- **バーコード連携**: USB接続スキャナーをブラウザのKeyboardイベントで取得（設備買い替え不要）
- **権限管理**: 部門×機能の2軸ロールベース管理（CanCanCanで実装）
- **在庫アラート**: 発注点を品目ごとに設定、下回ったらSlack通知 + メール通知
- **既存システム連携**: 販売管理システムのCSVを定時バッチで取り込み

## 成果

- **棚卸工数**: 2日間 → **4時間**（75%削減）
- **欠品発生件数**: 月平均15件 → **月2件以下**
- **在庫ロス**: 年間試算 **800万円削減**
- **リアルタイム在庫照会**: 電話確認ゼロ → 社内ポータルから即時確認
- **データ品質**: 担当者依存から脱却、入力精度99.7%（スキャナー入力により向上）
